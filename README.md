# ارزیابی فنی Product Software Engineer I

به مخزن ارزیابی فنی جایگاه **Product Software Engineer I (.NET)** خوش آمدید.

این آزمون با هدف ارزیابی توانمندی‌های فنی، تفکر محصول‌محور و کیفیت مهندسی طراحی شده است.

---

## مدت زمان پیشنهادی

**120 تا 180 دقیقه**

---

## هدف آزمون

در این ارزیابی انتظار می‌رود قابلیت **مدیریت درخواست اضافه‌کاری (Overtime Request Management)** برای یک محصول HRMS طراحی و پیاده‌سازی شود.

کارمندان باید بتوانند درخواست اضافه‌کاری ثبت کنند و مدیران بتوانند آن را تأیید یا رد نمایند. درخواست‌های تأییدشده باید برای فرآیند محاسبه حقوق و دستمزد آماده شوند.

---

## نکته مهم

این پروژه به‌صورت عمدی ناقص طراحی شده است.

برخی از اجزای اصلی دامنه در اختیار شما قرار نگرفته‌اند و بخشی از ارزیابی به توانایی شما در تحلیل، مدل‌سازی و طراحی این بخش‌ها اختصاص دارد.

شما مسئول طراحی و پیاده‌سازی موارد زیر هستید:

* موجودیت‌ها (Entities)
* مدل وضعیت‌ها (Status Model)
* ارتباط بین موجودیت‌ها
* قوانین کسب‌وکار
* محدودیت‌ها و Constraints
* ایندکس‌های پایگاه داده
* نحوه تغییر وضعیت‌ها (State Transitions)
* راهکارهای جلوگیری از خطاهای تکراری و مسائل مرتبط با Reliability

## نحوه تحویل

پس از تکمیل پیاده‌سازی:

1. یک Branch جدید ایجاد کنید.
2. تغییرات خود را Commit نمایید.
3. Pull Request ایجاد کنید.
4. به تمامی سوالات موجود در قالب Pull Request پاسخ دهید.

Pull Request باید شامل موارد زیر باشد:

* خلاصه تغییرات
* تصمیمات مدل‌سازی دامنه
* فرضیات محصول (Product Assumptions)
* ابهامات Requirement
* تصمیمات فنی
* تست‌های اضافه‌شده یا اصلاح‌شده
* محدودیت‌های فعلی راهکار
* پیشنهادهای توسعه در صورت داشتن زمان بیشتر
